# 隐私与责任声明

生效日期：2026 年 7 月 23 日

本声明适用于 RTProxyExchange 开源软件及其默认 Web 界面。任何公开或私有部署者都应根据实际运营主体、服务器所在地和适用法律补充自己的主体信息、联系方式与合规条款。

## 1. 请求处理方式

- 浏览器应用 API 只向当前站点的 `/api/*` 接口发送请求，Sub2API、CLIProxyAPI 与 Token 刷新请求由本站后端发起。
- 用户主动使用 OAuth 登录时，浏览器会打开 OpenAI 官方授权页面；这属于用户可见的页面导航。
- Refresh Token、Access Token、ID Token、API Key、管理密钥、管理员账号密码和账号 JSON 会在完成用户操作所需的时间内由后端处理。
- 后端会按用户指令将必要数据转发到 OpenAI 或用户自行填写的 Sub2API、CLIProxyAPI 地址。
- 本应用本身不建立用户数据库，也不会主动把上述敏感数据写入服务端文件。

## 2. 浏览器本地存储

以下数据可能保存在当前浏览器的 localStorage：

- 最近 20 条转换历史及完整转换结果，结果中可能包含 Token；
- 界面主题、导入模式等偏好；
- Sub2API 与 CLIProxyAPI 地址；
- 用户主动勾选“记住”后保存的 Admin Key 或 Management Key。

localStorage 中的数据通常为本机明文。使用共享设备、受控设备或不可信浏览器时，请勿启用“记住密钥”，并在使用后清空历史记录和站点数据。

Sub2API 管理员密码不会写入 localStorage；登录取得的 JWT 默认只保存在当前页面内存中。

## 3. 日志与基础设施

应用代码不会主动记录完整 Token 或管理密钥，进度和错误信息只应显示脱敏预览。但 Docker 日志、反向代理、CDN、云平台、防火墙或托管服务可能记录 IP 地址、时间、请求路径、User-Agent、响应状态等技术信息。

部署者应自行检查并配置基础设施日志，避免记录请求体、Authorization、Cookie、管理密钥或账号文件，并设置合理的访问权限与保留期限。

## 4. 第三方服务

当用户执行刷新、登录、上传或检查更新时，后端可能与以下服务通信：

- OpenAI 身份与 Token 接口；
- 用户指定的 Sub2API 实例；
- 用户指定的 CLIProxyAPI 实例；
- GitHub 的公开版本信息接口。

这些第三方服务按各自的隐私政策、日志策略和服务条款处理数据。项目维护者无法控制用户指定实例或第三方基础设施。

## 5. 安全边界

公网部署应至少启用 HTTPS、访问控制、强管理密钥、网络隔离和及时更新。不要把本服务直接暴露给不受信任用户，也不要在无法确认运营者身份的公共实例中提交真实凭据。

“不在服务端持久化”不等于请求内容从未进入服务器。敏感数据会在传输和处理期间经过后端内存及网络链路。

## 6. 用户责任

用户应保证：

- 对提交的账号、Token、密钥和目标服务拥有合法授权；
- 不使用本工具绕过访问控制、侵犯他人账号或违反第三方条款；
- 在提交数据前确认部署者、域名、HTTPS 证书和服务器可信；
- 妥善保管导出文件、下载文件、浏览器历史与本地存储。
